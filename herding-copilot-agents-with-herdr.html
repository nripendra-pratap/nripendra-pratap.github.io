Herding Copilot Agents with herdr 🐑

Jul 21, 2026 · 8 min read
AI AgentsCopilot CLITerminalOrchestration

Picture a farm. Not a metaphorical, hand-wavy farm. An actual one, with muddy boots and a thermos of tea. Each sheep is a GitHub Copilot CLI agent, quietly chewing through a task you handed it. The pasture is your terminal. And the border collie streaking across the field, keeping the whole flock together, is a little tool called herdr.

This is a field guide to running a flock of agents instead of a single, lonely one. It covers why you'd want to, why it beats a screen full of terminal windows, how to set it up, and a worked example where a watchdog keeps an eye on every sheep we let loose.

Life before a sheepdog

The first time you run an agent, you run one. You open a terminal, start copilot, describe a task, and watch it work. Lovely. One sheep, one field, no drama.

Then the work grows. You want one agent refactoring a module, another writing tests, a third chasing down a flaky build, and a fourth updating docs. So you do the obvious thing: you open more windows. A tab here. A split there. A second terminal app because you ran out of room. Within twenty minutes you're the farmer who let every gate swing open at once:

  • You can't see who's stuck. An agent finished five minutes ago, or it's blocked waiting for you to approve something, and you have no idea which window it's in.
  • Context-switching tax. You alt-tab through a maze, re-reading each pane to remember what it was even doing.
  • Lost sheep. Close the wrong window and a half-finished task is gone. Put your laptop to sleep and long-running agents wander off.
  • No coordination. The test-writing agent has no idea the refactor agent just finished. They can't hand work to each other.

One agent is a pet. A dozen agents in a dozen windows is a stampede. What you're missing isn't more windows. It's a sheepdog.

Why you need a herdr

herdr is an agent multiplexer that lives in your terminal. If you've used tmux, the shape is familiar: panes, tabs, sessions. But herdr is built specifically for AI agents, so it understands things tmux never could:

  • State at a glance. Every pane shows whether its agent is working, blocked (waiting on you), or done. That's the exact thing you were squinting to figure out manually.
  • Detach and walk away. Sessions survive disconnects and restarts. Close your laptop, reattach later from any terminal (even over SSH), and the flock is right where you left it.
  • Agents can drive herdr too. There's a plain socket API, so an agent can spawn its own panes, read another pane's output, and wait on a sibling to finish. This is the bit that makes our watchdog possible.
  • One small binary. No Electron, no browser tab pretending to be a terminal. It runs in whatever terminal you already use.
The mental model: you stop being a farmer who watches every sheep, and become one who glances at the dog. herdr does the watching; you make the decisions.

One pasture beats a dozen windows

"But I could just open more terminal windows." You could. Here's why the single-pasture approach wins once you have more than one or two agents:

  • You see the whole flock in one view. A sidebar lists every agent and its live status. No alt-tab archaeology to find the one that needs you.
  • Real terminal views, not a wrapper. herdr shows the agent's actual output, not a re-interpreted, screen-scraped copy that drifts out of sync.
  • Persistence is free. A pile of raw terminal windows dies with the app or the SSH connection. A herdr session keeps running and reattaches cleanly.
  • Agents coordinate. Separate windows are islands. Inside herdr, one agent can wait for another, read its logs, and pick up where it left off. That's the difference between a stampede and a flock.
  • Focus stays put. Spawning a new agent in a sibling pane doesn't yank your cursor away from what you're reading.

Setting up the farm

Install herdr (macOS/Linux):

# one-liner
curl -fsSL https://herdr.dev/install.sh | sh

# or Homebrew
brew install herdr

On Windows it's in beta:

powershell -ExecutionPolicy Bypass -c "irm https://herdr.dev/install.ps1 | iex"

Then walk out into the field where the work lives and start it:

cd ~/code/my-project
herdr

Split panes and run an agent in each one. For us that's the GitHub Copilot CLI, copilot. Detach any time with ctrl+b q; come back with herdr and the flock is still grazing.

The magic ingredient for coordination is the agent skill. It's a small instruction file that teaches an agent how to talk to herdr from inside a pane. Install it once, globally: 

npx skills add ogulcancelik/herdr --skill herdr -g

With the skill installed, any Copilot agent you start inside a herdr pane (where HERDR_ENV=1 is set) can split panes, run commands, read output, and wait on other agents. That's what turns a passive sheep into one that can call for the dog.

A day on the farm: naming the sheep

Let's actually herd something. We'll spin up a small flock, and since a nameless sheep is just livestock, we'll give each one a name and a job. (Yes, they're the Shaun the Sheep cast. No apologies.)

Start herdr, then use its CLI to lay out the pasture and put each sheep to work:

# Shaun (the clever one) refactors the auth module
herdr pane run w1:p1 "copilot -p 'Refactor src/auth into smaller modules, keep tests green'"

# Shirley (does the heavy lifting) writes the missing tests
herdr pane split w1:p1 --direction right
herdr pane run w1:p2 "copilot -p 'Add unit tests for the billing service, aim for 90% coverage'"

# Timmy (the little one) tidies up the docs
herdr pane split w1:p2 --direction down
herdr pane run w1:p3 "copilot -p 'Update the README and API docs to match the new auth module'"

# Nuts (a bit chaotic) chases the flaky integration test
herdr pane split w1:p3 --direction down
herdr pane run w1:p4 "copilot -p 'Find and fix the flaky test in tests/integration/checkout_spec'"

Four sheep, one field, each pane labelled and showing its own live status. You can peek at any of them without disturbing the others: 

# what has Nuts been up to?
herdr pane read w1:p4 --source recent --lines 50

Already this is calmer than four windows. But we're still the ones checking on them. Let's hire the dog.

Bitzer, the watchdog 🐕

Every good flock has a sheepdog. Ours is Bitzer, an agent whose entire job is to watch the other agents. Because herdr exposes its controls as a socket API (and the agent skill teaches Copilot how to use it), Bitzer isn't a special product feature. He's just another Copilot agent running in his own pane, pointed at the rest of the flock.

Give Bitzer his own pane and his marching orders:

herdr tab create --label watchdog
herdr pane run w1:p5 "copilot"

Then, inside Bitzer's session, the brief:

You are Bitzer, the watchdog for this herdr session.
You are running inside a herdr pane (HERDR_ENV=1), so use the herdr CLI.

Loop:
  1. List the panes and each agent's state (working / blocked / done).
  2. For any sheep that is DONE, read its last 40 lines and give me a
     one-line summary of what it produced.
  3. For any sheep that is BLOCKED waiting on input, raise a notification
     so the farmer (me) knows exactly which pane needs a decision.
  4. If a sheep has produced an error, note it and keep watching the rest.
Report as a compact status table. Don't touch the sheep's work, just watch.

Under the hood, Bitzer leans on the same primitives we used by hand. The difference is that he runs them, on a loop, so we don't have to: 

# wait until a specific sheep is finished grazing
herdr agent wait w1:p1 --until done

# read what it left behind
herdr pane read w1:p1 --source recent --lines 40

# nudge the farmer when a sheep is stuck at a gate
herdr notification show "Timmy (w1:p3) is blocked, needs your approval"
Now the loop closes. The sheep graze in parallel, Bitzer watches the field, and you watch Bitzer. When Shaun finishes his refactor, Bitzer summarises it and tells Shirley's tests to re-run. When Nuts wanders into an error, you get one tidy notification instead of discovering it an hour later in a window you forgot you had open.

The takeaway

Running one agent is easy. The moment you want several working at once, the bottleneck stops being the agents and becomes you: your attention, spread thin across a wall of terminals. herdr fixes that by giving the flock a single pasture, a live status board, sessions that don't wander off, and an API so the agents can look after each other.

You don't need to watch every sheep. You just need a good dog. 🐑🐕
